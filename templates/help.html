User Manual

Admiralty is the FOIA workflow management system in the DockLabs suite. It moves a Freedom of Information Act request through nine well-defined statuses — from intake to scope definition, search, attorney review, response packaging, and (when needed) appeal — with statutory deadline tracking, exemption claims, and a full immutable audit trail at every step.

In standalone mode, Admiralty searches its own document repository. When deployed alongside Beacon CRM, it also reaches into Beacon's interaction and note corpus for cross-product responsive-record discovery.

Nine statuses, in order:

  1. Received — intake complete.
  2. Scope Defined — search parameters set.
  3. Searching — finding responsive records.
  4. Under Review — attorney determining responsiveness and exemptions.
  5. Package Ready — compiled package awaiting senior review.
  6. Senior Review — final approval pass.
  7. Responded — response sent to requester.
  8. Appealed (optional) — requester has filed an appeal.
  9. Closed — terminal.

Connecticut FOIA (CGS §1-206) gives an agency 4 business days from receipt to acknowledge a request. Federal FOIA (5 USC 552) gives 20 business days. Admiralty stores three deadline-relevant fields:

  • date_received — when the agency got the request. Triggers the clock.
  • statutory_deadline — computed deadline. Indexed; powers the dashboard Overdue count and the Awaiting Me column in Helm.
  • extended_deadline — set when an extension has been negotiated with the requester.

From a request detail page, click Define Scope. A scope carries:

  • Date range — start and end of the search window.
  • Keywords — one term per line.
  • Company names / Contact names — one per line (Beacon mode).
  • Record types — which kinds of records to include (notes, interactions, documents).
  • Scope notes — free-text description of the search strategy.

Saving a scope on a received request automatically advances it to scope_defined.

It depends on deployment mode:

  • Standalone Admiralty — searches the FOIADocument repository (uploaded files with extracted text content).
  • Beacon-embedded — also reaches into Beacon's Interaction and Note rows in Zone 1 (shared public records) and Zone 2 (DECD internal). Zone 3 (act_private) is never returned — those records are not FOIA-responsive by policy.

Each match becomes a FOIASearchResult with a content snapshot frozen at search time so it doesn't drift if the source is later edited.

Likely Responsive
Matches the scope and appears releasable.
Likely Exempt
Matches but contains content covered by a statutory exemption.
Needs Review
Match found; classification deferred to an attorney.
Not Relevant
False positive — included in the audit trail for completeness.

From the search results page, click into a result and write a determination. Fields:

  • Decision — Release / Withhold / Partial Release / Refer to Another Agency.
  • Exemptions claimed — multi-select from the active StatutoryExemption table; required for Withhold or Partial Release.
  • Justification — legal reasoning for the audit trail and privilege log.
  • Redacted content — sanitized version of the record (Partial Release).
  • Attorney notes — optional internal notes.

Saving stamps the reviewing attorney and the timestamp. The request stays in Under Review until every result has a determination and the package has been compiled.

Click AI Review on the search results page. Admiralty asks Claude to read every result and flag classification mismatches:

  • should_release — currently marked Withhold, AI thinks it's responsive and not exempt.
  • should_withhold — currently marked Release, AI sees likely-exempt content.
  • review_recommended — AI is high-confidence but the human classification is still needs_review.
  • ok — AI agrees with the current classification.

AI review is advisory. Every determination still requires an attorney's sign-off. The feature is silent when ANTHROPIC_API_KEY isn't configured.

From a request in Under Review, click Compile Response. Admiralty rolls up every determination into a FOIAResponsePackage with counts (released / withheld / partially released) and slots for the cover letter, response file, and privilege log.

Compiling advances the request to Package Ready. From there a manager submits to Senior Review, and the senior reviewer either sends the response (→ Responded) or returns it to review with a comment.

From a Responded request, click Record Appeal. The appeal carries:

  • Appeal number, filed date, status (Filed / Hearing Scheduled / etc.).
  • Appellant arguments, agency response, hearing date and notes.
  • Decision date, decision summary, and an uploaded decision document.
  • Lessons learned for future requests.

Visit Documents in the sidebar. Upload one file at a time at /foia/documents/upload/, or drag-and-drop multiple at /foia/documents/bulk-upload/.

Every upload runs through FileSecurityValidator (extension allowlist, magic-byte check, optional ClamAV) and is capped at KEEL_MAX_UPLOAD_SIZE (default 10 MB). Documents become searchable as soon as their content is extracted.

Admiralty publishes two endpoints to Helm:

  • /api/v1/helm-feed/ — aggregate counts for the fleet metric grid.
  • /api/v1/helm-feed/inbox/?user_sub=<oidc_sub> — per-user inbox. Returns FOIA requests assigned to that user where the status is open, carrying the statutory deadline as the due date so Helm's Awaiting Me column can sort by urgency.

Visit /notifications/preferences/ (also reachable from the sidebar user-menu dropdown).

For each event type, toggle the in-app and email channels independently. Email delivery uses Resend in production.
